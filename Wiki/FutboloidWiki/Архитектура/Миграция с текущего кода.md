## Сейчас

| Файл | Проблема |
|------|----------|
| `MainMenu.unity` + `MainMenuController` | Отдельная сцена — **против** целевой архитектуры |
| `GameManager` singleton | Смешивает таймер, счёт, state, `LoadScene` |
| `SampleScene` | Станет `Game.unity`, грузится additive |
| Build: MainMenu → SampleScene | Нужен только `Bootstrap` |

## План миграции (этапы)

### Этап 1 — Bootstrap + Root

- [ ] Добавить `Bootstrap.unity`, пакет VContainer + UniTask
- [ ] `GameStartupHandler`, `GameDirector`, `AppRootState` (минимум)
- [ ] Build settings: только Bootstrap
- [ ] RootScene + пустой root scope

### Этап 2 — Game additive

- [ ] Переименовать `SampleScene` → `Game`
- [ ] `AppGameState` грузит сцену additive
- [ ] `ObjectRegistry` или точечная регистрация view — см. [[Связь сцены с кодом]]
- [ ] Перенести UI HUD в overlay prefab

### Этап 3 — FSM

- [ ] `OverlayStateController`: MainMenu / OnField / Pause
- [ ] `PitchStateMachine`: KickoffWait / Simulating / …
- [ ] Разобрать `GameManager` на `MatchFlow` + `PitchStateMachine`

### Этап 4 — Main Menu overlay

- [ ] Удалить `MainMenu.unity` из билда
- [ ] Prefab `MainMenuWidget` + лидерборд
- [ ] `BotSimulationController` для фона

### Этап 5 — Polish

- [ ] Scene transition (турнир)
- [ ] `ISaveStorage` + лидерборд
- [ ] Tournament overlay

## Маппинг GameManager → новые классы

| Было в GameManager | Куда |
|--------------------|------|
| `GameState` enum | `PitchStateMachine` |
| Таймер матча | `MatchFlow` |
| Счёт голов | `MatchFlow` |
| `RecordGoal` | событие → Pitch FSM → Reshuffle |
| `StartMatch` LoadScene | убрать — сцена уже загружена |
| Singleton Instance | убрать — DI |
