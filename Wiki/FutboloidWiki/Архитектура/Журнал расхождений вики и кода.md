# Журнал расхождений вики и кода

← [[Индекс архитектуры]]

Таблица **устаревших** мест вики и как они приведены в соответствие с проектом. Обновлять при крупных изменениях кода.

**Последняя сверка:** 2026-07 (генерация врагов, вайп-победа, сцена `Game.unity`).

## Всё ещё расходится (осознанно)

| Тема | Вики | Код | План |
|------|------|-----|------|
| `PassToNearest`, заряд паса | [[Враги и защитники]], [[Движение мяча]] | Не реализовано | P1 |
| `RunStateService`, перки | [[Прогрессия и эффекты]] | Нет в DI | MVP забега позже |
| `StatusEffectService` | Документирован | Нет в коде | P1 |
| Reshuffle анимация | Полное описание | `DefenderReshuffleService` с твинами — **есть**; в «Враги» было «заглушка» | Обновить статус в [[Враги и защитники]] при следующей правке |
| Главное меню | GDD: отдельная сцена | Оверлей на Root | Уже в [[Индекс архитектуры#Расхождение с GDD]] |
| `matchesToWin` | Цель 10+ | Default **3** в `GameplaySettings` | Поднять когда будет сетка |
| `MatchEndReason` enum | В примерах кода вики | `bool PlayerWon` | Не вводить enum без нужды; вики → `PlayerWon` |
| Визуал архетипов | Разные спрайты по роли | Один спрайт | После генератора |
| Динамическая сложность | Задел в §8 | Нет | Post-MVP |
| Comeback в турнире | §8 GDD | `Eliminated` окончательно | Отдельная задача |

## Как пользоваться

1. Перед крупным рефакторингом — пройтись по таблице «Всё ещё расходится».  
2. После мержа фичи — строка в «Исправлено» + правка связанных статей.  
3. Новые системы — сначала заметка в [[Генерация врагов]] / GDD, потом код (или наоборот, но журнал обновить).
