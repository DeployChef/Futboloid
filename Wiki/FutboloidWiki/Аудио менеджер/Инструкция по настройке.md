# Инструкция по настройке звуков

> [!date] Обновлено
> 30.06.2026

---

## Быстрый старт

### 1. Открой сцену `Game.unity`

В Hierarchy найди объект `AudioManager`.

### 2. Создай динамики

Каждый тип звука — это отдельный дочерний объект внутри `AudioManager`.

**Шаги:**
1. Правой кнопкой на `AudioManager` → `Create Empty`
2. Назови по имени события: `Speaker_BallHit`, `Speaker_Goal`, `Speaker_Start`
3. На каждый объект повесь скрипт `AudioClipSource`

### 3. Настрой каждый динамик

В Inspector каждого `AudioClipSource` заполни:

| Поле | Что делать |
|------|------------|
| **Event Name** | Имя события (например, `BallHit`) |
| **Clips** | Перетащи массив звуков (например, 5 ударов) |
| **Mixer Group** | Выбери группу микшера (например, `SFX`) |

### 4. Проверь `AudioManagerLifetimeScope`

- Должен быть на сцене `Game.unity`
- `Auto Run` должен быть **включён** (галочка в Inspector)

## Каталог событий для настройки

| Динамик | Event Name | Описание | Событие в коде |
|---------|------------|----------|----------------|
| `Speaker_BallHit` | `BallHit` | Удар о стену/защитника/вратаря | `BallHitEvent`, `BallReturnedToKeeperEvent` |
| `Speaker_Goal` | `GoalScored` | Гол забит | `GoalScoredEvent` (isPlayerGoal=true) |
| `Speaker_GoalConceded` | `GoalConceded` | Гол пропущен | `GoalScoredEvent` (isPlayerGoal=false) |
| `Speaker_Start` | `MatchStart` | Свисток перед матчем | `BallServedEvent` |
| `Speaker_End` | `MatchEnd` | Свисток конца матча | `MatchEndedEvent` |

## Как добавить новый звук

1. Создай дочерний объект в `AudioManager`
2. Назови: `Speaker_НовоеИмя`
3. Повесь `AudioClipSource`
4. Заполни поля:
   - **Event Name:** `НовоеИмя`
   - **Clips:** перетащи звуки
   - **Mixer Group:** выбери группу
5. В коде подпишись на событие в `AudioManager.OnEnable()`:
   ```csharp
   _bus.Subscribe<НовоеСобытие>(e => PlayEvent("НовоеИмя"));
   ```

## Проверка работы

1. Запусти игру
2. Ударь мячом о стену → должен прозвучать случайный звук из `Speaker_BallHit`
3. Забей гол → звук из `Speaker_Goal`
4. Проверь громкость в `AudioMixer`

## Частые проблемы

| Проблема | Решение |
|----------|---------|
| Звуки не играют | Проверь `Auto Run` у `AudioManagerLifetimeScope` |
| Ошибка компиляции | Убедись, что файлы в `Futboloid.Core` |
| Звук в общем канале | Проверь `Mixer Group` в `AudioClipSource` |
| Слишком много звуков одновременно | Увеличь `Max Voices` в `AudioManager` |
