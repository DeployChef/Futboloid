> [!warning] ВНИМАНИЕ

## 📋 Что мы делаем

Мы создаём систему звука в игре, которая:
- ✅ Играет звуки ударов мяча
- ✅ Играет звуки голов
- ✅ Играет фоновую музыку
- ✅ Плавно включает/выключает музыку (fade in/out)
- ✅ Не позволяет звукам мешать друг другу (приоритеты)

---

## 🗂️ Где находятся файлы

Все скрипты аудио-системы лежат здесь:

| Файл | Путь в проекте | Что делает |
|------|----------------|------------|
| `AudioManager.cs` | `Assets/_Projects/Code/Futboloid.Core/AudioManager.cs` | Главный мозг. Управляет всеми звуками |
| `AudioClipSource.cs` | `Assets/_Projects/Code/Futboloid.Core/AudioClipSource.cs` | Конкретный динамик (гол, удар, музыка) |

> [!tip] Как найти файлы
> 1. В Unity откройте окно **Project** (обычно внизу)
> 2. Раскройте папку `Assets`
> 3. Затем `_Projects` → `Code` → `Futboloid.Core`
> 4. Там увидите нужные файлы

## 🎯 Часть 1: Настройка AudioManager (Мозг)

### Шаг 1.1: Найдите AudioManager на сцене

1. Откройте сцену `Game.unity` (папка `_Projects/Scenes/Game.unity`)
2. В окне **Hierarchy** (слева) найдите объект с именем `AudioManager`
3. Если его нет — создайте:
   - Правой кнопкой в Hierarchy → `Create Empty`
   - Назовите его `AudioManager`
   - Добавьте компонент: `Add Component` → `AudioManager`

### Шаг 1.2: Настройте глобальные параметры

Выделите `AudioManager` и посмотрите в **Inspector** (справа):

| Поле | Значение | Что делает |
|------|----------|------------|
| **Max Voices** | `8` | Максимум звуков одновременно. 8 — достаточно |
| **Enable Fade** | ✅ Включено | Включает плавное появление/затухание звуков |

> [!info] Что такое Fade?
> Fade — это плавное увеличение/уменьшение громкости. Вместо резкого "БАМ" звук нарастает за 1 секунду. Приятнее для ушей.

### Шаг 1.3: Проверьте работу

AudioManager — это **синглтон**. Это значит:
- Он один на всю игру
- Вызывается так: `AudioManager.Instance.PlayEvent("BallHit")`
- Не требует никакой настройки DI или подключений

## 🔊 Часть 2: Создание динамиков (Конкретные звуки)

Каждый тип звука — это отдельный **динамик**. Динамик — это дочерний объект AudioManager.

### Шаг 2.1: Создайте первый динамик — Удар мяча

1. Выделите `AudioManager` в Hierarchy
2. Правой кнопкой → `Create Empty`
3. Назовите: `Speaker_BallHit`
4. С этим объектом должно быть повешено что-то. Если нет:
   - `Add Component` → `AudioClipSource`

### Шаг 2.2: Настройте динамик удара

Выделите `Speaker_BallHit` и заполните Inspector:

#### Блок "Настройки"

| Поле | Что вписать | Зачем |
|------|-------------|-------|
| **Event Name** | `BallHit` | Название события. В коде будет вызов `PlayEvent("BallHit")` |
| **Clips** | Перетащите 3-5 звуков ударов | Звук будет выбираться случайный из этого массива |

> [!howto] Как перетащить звуки
> 1. В Project окне найдите папку с звуками (`Assets/_Projects/Audio/SFX/Ball/`)
> 2. Перетащите 3-5 файлов из Project в поле `Clips` в Inspector
> 3. Растяните количество элементов (Number) до нужного значения

#### Блок "Кулдаун"

| Поле | Значение | Зачем |
|------|----------|-------|
| **Cooldown** | `0.3` | Минимум 0.3 сек между ударами. Чтобы звуки не сливались в кашу |

#### Блок "Loop"

| Поле | Значение | Зачем |
|------|----------|-------|
| **Loop** | `False` | Удар не должен играть циклично. Это разовый звук |

#### Блок "Fade In/Out"

| Поле | Значение | Зачем |
|------|----------|-------|
| **Enable Fade** | `False` | Удары не нужно плавно включать. Они и так короткие |
| **Fade Duration** | `0.1` | На всякий случай. Для ударов не важно |

#### Блок "Микшер"

| Поле | Что вписать | Зачем |
|------|-------------|-------|
| **Mixer Group** | Перетащите группу `SFX` из микшера | Чтобы звук попадал в правильную группу и был виден в индикаторе |

> [!howto] Как найти группу микшера
> 1. В Project окне найдите файл `MainMixer.mixer` (обычно в `Assets/_Projects/Audio/Mixers/`)
> 2. Дважды кликните, откроется окно **Audio Mixer**
> 3. Найдите группу `SFX`
> 4. Перетащите её из окна микшера в поле `Mixer Group` в Inspector

### Шаг 2.3: Создайте динамик для гола

Повторите шаги 2.1-2.2:

1. Правой кнопкой на `AudioManager` → `Create Empty`
2. Назовите: `Speaker_Goal`
3. Добавьте компонент `AudioClipSource`

Настройка:

| Поле | Значение | Зачем |
|------|----------|-------|
| **Event Name** | `GoalScored` | Событие при голе |
| **Clips** | 2-3 звука гола | Случайный выбор |
| **Cooldown** | `1.0` | Голы не так часто, можно подождать |
| **Loop** | `False` | |
| **Enable Fade** | `True` | Гол можно включать плавно |
| **Fade Duration** | `0.5` | Быстрый фейд |
| **Mixer Group** | `SFX` | |

### Шаг 2.4: Создайте динамик для пропущенного гола

1. `Create Empty` → `Speaker_GoalConceded`
2. Добавьте `AudioClipSource`

Настройка:

| Поле | Значение |
|------|----------|
| **Event Name** | `GoalConceded` |
| **Clips** | 2-3 звука пропущенного гола |
| **Cooldown** | `1.0` |
| **Loop** | `False` |
| **Enable Fade** | `True` |

### Шаг 2.5: Создайте динамик для свистка

1. `Create Empty` → `Speaker_Start`
2. Добавьте `AudioClipSource`

Настройка:

| Поле | Значение |
|------|----------|

| **Clips** | 1 звук свистка |
| **Cooldown** | `0.5` |
| **Loop** | `False` |
| **Enable Fade** | `True` |
| **Fade Duration** | `0.3` |
| **Mixer Group** | `SFX` |

### Шаг 2.6: Создайте динамик для конца матча

1. `Create Empty` → `Speaker_End`
2. Добавьте `AudioClipSource`

Настройка:

| Поле | Значение |
|------|----------|
| **Event Name** | `MatchEnd` |
| **Clips** | 1 звук финального свистка |
| **Cooldown** | `0` |
| **Loop** | `False` |
| **Enable Fade** | `True` |
| **Fade Duration** | `1.0` |
| **Mixer Group** | `SFX` |

### Шаг 2.7: Создайте динамик для музыки

1. `Create Empty` → `Speaker_Music`
2. Добавьте `AudioClipSource`

Настройка:

| Поле | Значение | Зачем |
|------|----------|-------|
| **Event Name** | `MusicStart` | Событие запуска музыки |
| **Clips** | 1-2 музыкальных трека | Будет выбран случайный |
| **Cooldown** | `0` | Музыка не имеет кулдауна |
| **Loop** | `True` ⚠️ **ВАЖНО** | Музыка играет бесконечно |
| **Enable Fade** | `True` | Музыка включается/выключается плавно |
| **Fade Duration** | `1.5` | Длинный фейд для плавности |
| **Mixer Group** | `Music` | Музыка в отдельной группе |

> [!warning] Loop = True
> Для музыки ОБЯЗАТЕЛЬНО установите Loop в `True`. Иначе музыка проиграет один раз и остановится.

## 🔗 Часть 4: Как код вызывает звуки

В игровых скриптах звуки вызываются одной строкой:

```csharp
// Пример из BallMotion.cs (движение мяча)
AudioManager.Instance.PlayEvent("BallHit");

// Пример из MatchFlow.cs (конец матча)
AudioManager.Instance.PlayEvent("MatchEnd");

// Пример из MatchFlow.cs (старт музыки)
AudioManager.Instance.PlayEvent("MusicStart");
```

> [!tip] Как добавить новый звук
> 1. Создайте динамик в Unity (как в Части 2)
> 2. Назовите Event Name, например `Click`
> 3. В коде добавьте вызов: `AudioManager.Instance.PlayEvent("Click")`
> 4. Всё! Больше ничего делать не нужно.

## 🎛️ Часть 5: Приоритеты (Как звуки не мешают друг другу)

### Проблема
Что если играют 10 звуков одновременно? Музыка, удары, голы — всё сливается.

### Решение: Приоритеты

Каждый динамик имеет поле **Priority**:
- `0` = Высший приоритет (гол, музыка, свисток)
- `50` = Средний приоритет (UI, бонусы)
- `100` = Низкий приоритет (удары мяча — их много)

**Правило:** Если лимит голосов (8) исчерпан, новый звук с высоким приоритетом вытеснит звук с низким приоритетом.

### Настройка приоритетов

| Звук | Приоритет | Почему |
|------|-----------|--------|
| Музыка | `0` | Не должна прерываться |
| Гол | `0` | Критически важно |
| Свисток | `5` | Важно |
| Бонус | `10` | Заметно |
| UI клик | `40` | Средне |
| Удар мяча | `80` | Часто, не важно если прервётся |

## 🐛 Часть 6: Частые проблемы и решения

### Проблема 1: Звуки не играют

**Причины:**
1. Динамик не создан в Unity
2. Event Name не совпадает с тем, что в коде
3. Clips пустой

**Решение:**
- Проверьте, что динамик существует в Hierarchy
- Проверьте, что Event Name точно совпадает (регистр важен!)
- Перетащите звуки в поле Clips

### Проблема 2: Нет индикации на микшере

**Причина:** Не назначена Mixer Group

**Решение:**
1. Откройте AudioMixer (двойной клик на `.mixer` файле)
2. Найдите группу `SFX` или `Music`
3. Перетащите группу в поле Mixer Group динамика

### Проблема 3: Музыка не играет циклично

**Причина:** Loop = False

**Решение:**
- Убедитесь, что у `Speaker_Music` поле `Loop` установлено в `True`

### Проблема 4: Звуки сливаются в кашу

**Причина:** Слишком маленький Cooldown

**Решение:**
- Увеличьте Cooldown у `Speaker_BallHit` до `0.5`
- Убедитесь, что Allow Overlap = False для голосовых звуков

### Проблема 5: Fade не работает

**Причины:**
1. Enable Fade выключен в AudioManager
2. Enable Fade выключен в конкретном динамике

**Решение:**
- Включите Enable Fade в AudioManager
- Включите Enable Fade в нужных динамиках

## 🧠 Часть 8: Как всё работает (простыми словами)

```
1. Игрок бьёт по мячу
2. Код в BallMotion.cs видит удар и пишет:
   AudioManager.Instance.PlayEvent("BallHit")
3. AudioManager ищет динамик с Event Name = "BallHit"
4. Находит Speaker_BallHit
5. Выбирает случайный звук из Clips
6. Воспроизводит его через AudioSource
7. Звук попадает в группу SFX микшера
8. Вы слышите звук! 🎧
```

**Fade In:** Звук не появляется резко. Громкость нарастает от 0 до 100% за 1 секунду.

**Fade Out:** Звук не обрывается резко. Громкость падает от 100% до 0% за 1 секунду, потом звук останавливается.

**Приоритеты:** Если играет 8 звуков и новый звук важнее — менее важный звук останавливается.

**Cooldown:** Один и тот же звук не может играть чаще, чем раз в N секунд.

**Loop:** Если включено, звук повторяется бесконечно (для музыки).
