# Контекст чата — Аудио рефакторинг (05.07.2026)

## Решение

Убрать `AudioManager` синглтон и `AudioClipSource` на Game-сцене. Перейти на:

- `IGameEventBus` — триггеры звука
- `AudioCatalog` (SO) — CMS клипов
- `AudioService` (App scope) — подписки и правила
- `AudioPlaybackHost` (Root scene) — пул AudioSource + музыкальный канал

## Почему не два менеджера (меню / игра)

Один сервис + mixer groups + `NavigationChangedEvent` для pause/resume. Масштаб проекта не требует отдельных классов.

## Новое событие

`MatchStartedEvent` — публикует `MatchFlow`, когда таймер матча пошёл на поле. Аудио: свисток + музыка. Не путать с `BallServedEvent` (физика мяча — RunState, PitchFSM).

## Удалено

- `AudioManager.cs`
- `AudioClipSource.cs`
- `RandomMusicPlayer.cs`
- `BallHitEvent.cs` (не использовался)
- Прямые `AudioManager.Instance` из BallMotion, MatchFlow, RunStateService, OverlayStateController

## Настройка на сцене (ручная)

1. Root.unity → `Audio` + `AudioPlaybackHost`
2. Заполнить `AudioCatalog.asset`
3. Game.unity → удалить `AudioManager`
