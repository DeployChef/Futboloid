## Архитектура аудио-системы

### Компоненты

```
AudioManager (синглтон)
├── Speaker_BallHit (AudioClipSource)
│   ├── eventName: "BallHit"
│   ├── clips: [удар1.wav, удар2.wav, удар3.wav]
│   ├── priority: 80
│   ├── cooldown: 0.3
│   ├── loop: false
│   ├── enableFade: false
│   └── mixerGroup: SFX
├── Speaker_Goal (AudioClipSource)
│   ├── eventName: "GoalScored"
│   ├── priority: 0
│   └── ...
├── Speaker_Music (AudioClipSource)
│   ├── eventName: "MusicStart"
│   ├── loop: true
│   ├── enableFade: true
│   ├── fadeDuration: 1.5
│   └── mixerGroup: Music
└── ...
```

### Логика работы

```
Игровой процесс
  ↓
Событие (удар, гол, подача)
  ↓
Код вызывает: AudioManager.Instance.PlayEvent("Имя")
  ↓
AudioManager ищет динамик по имени
  ↓
Проверка:
  1. Allow Overlap — можно ли играть?
  2. Cooldown — не слишком ли часто?
  3. Priority — достаточно ли важен?
  4. Max Voices — есть ли место?
  ↓
Если всё ОК:
  1. PlayRandom() — выбирает случайный звук
  2. FadeIn() — плавно включает (если включено)
  3. source.Play() — воспроизводит
  ↓
Звук попадает в AudioMixer (SFX или Music)
```

### Вызовы из кода

```csharp
// В BallMotion.cs — при ударе мяча
AudioManager.Instance.PlayEvent("BallHit");

// В BallMotion.cs — при голе
AudioManager.Instance.PlayEvent("GoalScored");

// В MatchFlow.cs — при подаче мяча (старт таймера)
AudioManager.Instance.PlayEvent("MusicStart");

// В MatchFlow.cs — при рестарте матча
AudioManager.Instance.StopEvent("MusicStart");

// В MatchFlow.cs — при конце матча
AudioManager.Instance.PlayEvent("MatchEnd");
AudioManager.Instance.StopEvent("MusicStart");

// В OverlayStateController.cs — при выходе в меню
AudioManager.Instance.PauseMusic();

// В OverlayStateController.cs — при возврате в игру
AudioManager.Instance.ResumeMusic();
```

## Ключевые решения

### Почему AudioManager — синглтон, а не через DI?
- Проще для новичков
- Не нужно настраивать LifetimeScope
- Не нужно регистрировать в DI
- Просто: `AudioManager.Instance.PlayEvent("BallHit")`

### Почему динамики — дочерние объекты?
- Визуально понятно в Hierarchy
- Легко добавлять новые звуки
- Не нужно менять код для добавления звука
- Настройка в Inspector, а не в коде

### Почему Fade In/Out через корутины?
- Плавное изменение громкости в Unity требует обновления каждый кадр
- Корутина `IEnumerator` идеально подходит для этого
- Можно остановить в любой момент через `StopCoroutine()`

### Почему музыка останавливается при Reset()?
- Чтобы не играть фоновая музыка между раундами
- Музыка должна начинаться только при подаче мяча
- При рестарте матча таймер сбрасывается, музыка тоже

### Почему пауза музыки через Pause()/Resume(), а не Stop()/Play()?
- `Pause()` и `Resume()` сохраняют позицию воспроизведения
- `Stop()` + `Play()` начинали бы трек с начала
- Игрок ожидает, что музыка продолжится с того же места

### Почему логика паузы в OverlayStateController?
- `OverlayStateController` управляет переключением состояний навигации
- Он знает, когда игрок выходит в меню и когда возвращается
- `AudioManager` не должен знать о навигации — он просто предоставляет методы

| Событие в коде | Event Name | Динамик | Где вызывается |
|----------------|------------|---------|----------------|
| `BallHitEvent` | `BallHit` | `Speaker_BallHit` | `BallMotion.ResolveHit()` |
| `BallReturnedToKeeperEvent` | `BallHit` | `Speaker_BallHit` | `BallMotion.ResolveHit()` |
| `GoalScoredEvent (isPlayerGoal=true)` | `GoalScored` | `Speaker_Goal` | `BallMotion.TryScoreGoal()` |
| `GoalScoredEvent (isPlayerGoal=false)` | `GoalConceded` | `Speaker_GoalConceded` | `BallMotion.TryScoreGoal()` |

| `MatchEndedEvent` | `MatchEnd` | `Speaker_End` | `MatchFlow.EndMatch()` |

| `PerkPickedEvent` | `PerkPick` | `Speaker_Perk` | `RunStateService.ApplyPerkPick()` |
| `NavigationChangedEvent` | — | — | `OverlayStateController.SetState()` (пауза/возобновление музыки) |
