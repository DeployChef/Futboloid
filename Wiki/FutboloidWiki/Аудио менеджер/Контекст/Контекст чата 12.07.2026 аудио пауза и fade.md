## Проблемы

### Проблема 1: Наложение треков при выходе из паузы

**Симптом:** После выхода из паузы музыкальный трек (`MusicMatch`) продолжал играть с того же места, но к нему добавлялся **новый** трек. Два трека играли одновременно.

**Корневая причина:** Метод `ResumeMusic()` вызывал `FadeAndStopMusicAsync()` вместо `UnPause()`. Он **останавливал** музыку полностью, а не возобновлял её.

### Проблема 2: Дублирование трека паузы

**Симптом:** Трек паузы (`UiPauseOpen`) продолжал играть после выхода из паузы, накладываясь на основной игровой трек.

**Корневая причина:** Трек паузы запускался **дважды**:
1. `PauseMusic()` → `PlayPauseSound()` → `_pauseSource` (отдельный AudioSource для трека паузы)
2. `AudioService.OnNavigationChanged()` → `Play("UiPauseOpen")` → пул `_uiVoices` (дублирующий экземпляр)

При выходе из паузы `ResumeMusic()` останавливал только `_pauseSource`, а дублирующий трек из пула продолжал играть.

### Проблема 3: Неправильный порядок операций при паузе/возобновлении

**Симптом:** Трек паузы не ставился на паузу одновременно с музыкальным треком, создавая ощущение "рассинхрона".

## Решения

### Решение 1: Исправить `ResumeMusic()`

**Файл:** `Assets/_Projects/Code/Futboloid.Main/Audio/AudioManager.cs`

**Было:**
```csharp
public void ResumeMusic()
{
    _musicPaused = false;
    _musicSource.UnPause();
    // ...
    FadeAndStopMusicAsync(_musicFadeOutDuration, _musicFadeCts.Token).Forget(); // ❌ Останавливает музыку
    _musicSoundId = null;
}
```

**Стало:**
```csharp
public void ResumeMusic()
{
    // Сначала ставим трек паузы на паузу (мгновенно)
    if (_pauseSource != null && _pauseSource.clip != null && _pauseSource.isPlaying)
        _pauseSource.Pause();

    // Затем возобновляем музыку
    _musicPaused = false;
    _musicSource.UnPause();

    // Фэйд ин музыки
    CancelMusicFade();
    _musicFadeCts = new CancellationTokenSource();
    var targetVolume = _musicVolumeBeforePause > 0f ? _musicVolumeBeforePause : 1f;
    FadeToAsync(_musicSource, targetVolume, _musicFadeDuration, _musicFadeCts.Token).Forget();
}
```

## Итоговая логика работы

### Переход на паузу (Escape)

```
1. PauseMusic() вызывается из AudioService
2. SFX с stopOnPause=true → мгновенная остановка
3. MusicMatch → FadeOut (fadeOutDuration сек) → Pause()
4. UiPauseOpen → Play() → FadeIn (fadeDuration сек)
```

### Выход из паузы (Escape)

```
1. ResumeMusic() вызывается из AudioService
2. UiPauseOpen → Pause() (мгновенно)
3. MusicMatch → UnPause() → FadeIn (fadeDuration сек до prevVolume)
```

### Полный выход на поле (запуск нового трека)

```
1. PlayMusic() вызывается из HandleFieldMusic()
2. _pauseSource → Stop() (полная остановка)
3. MusicMatch → Play() → FadeIn (fadeDuration сек)
```
