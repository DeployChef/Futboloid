# Контекст чата — Аудио рефакторинг (05–08.07.2026)

## Итоговая архитектура (08.07.2026)

- `IGameEventBus` — триггеры звука
- `AudioService` (App scope) — подписки, передаёт **Sound Id** в менеджер
- `AudioManager` (Root scene) — конфиг `AudioCatalog`, cooldown, overlap, пул, музыка
- `IAudioManager` — интерфейс для DI (сервис + прямые вызовы из UI)

```
События → AudioService → IAudioManager.Play("Id")
UI/кнопки → IAudioManager.Play("Id") напрямую
              ↓
         AudioManager (config)
```

## Ключевые решения

- Геймплей не вызывает `AudioSource` напрямую
- Конфиг (`AudioCatalog`) только у `AudioManager`, не в App DI
- Музыка стартует при входе на `OnField`, не при подаче мяча
- `MatchStartedEvent` → только свисток `MatchStart`
- Пауза музыки: fade out + `Pause()`, resume с того же `time`
- `BallHit` (стена) / `BallHitMan` (вратарь, защитник) — разные Sound Id
- Подписки на `ComboScoreChangedEvent`, `StatusEffectApplied/Removed`

## Удалено / переименовано

- `AudioManager` синглтон на Game (legacy)
- `AudioPlaybackHost` → **`AudioManager`**
- `IAudioPlayback` → **`IAudioManager`**
- `maxSfxVoices` из `AudioCatalog`

## Настройка на сцене (ручная)

1. Root.unity → `Audio` + `AudioManager`, поле **Config** → `AudioCatalog.asset`
2. Заполнить `AudioCatalog.asset`
3. Game.unity → удалить legacy `AudioManager` / `Speaker_*`
