## 6.1 Пошаговый чек-лист

### Для новичка: как начать

**День 1-2: Знакомство с проектом**
- [ ] Открой Unity и запусти проект
- [ ] Открой сцену `Root.unity` — найди `Startup` и `RootLifetimeScope`
- [ ] Открой сцену `Game.unity` — найди `BallView`, `GoalkeeperView`, `DefenderSpawner`
- [ ] Запусти игру в Play Mode и пройдись по всем экранам

**День 3-4: Архитектура**
- [ ] Прочитай `Startup.cs` — пойми, как запускается игра
- [ ] Прочитай `GameDirector.cs` — пойми, как переключаются экраны
- [ ] Прочитай `GameEventBus.cs` — пойми шину событий
- [ ] Найди все `Publish()` и `Subscribe()` в проекте

**День 5-6: Аудио-система**
- [ ] Открой `AudioManager.cs` — пойми пулы и фейды
- [ ] Открой `AudioService.cs` — пойми маппинг событий
- [ ] Открой `AudioCatalog.asset` — найди `BallHit`, `MusicMatch`
- [ ] Попробуй добавить новый звук

**День 7+: Геймплей**
- [ ] Прочитай `BallMotion.cs` — пойми физику мяча
- [ ] Прочитай `MatchFlow.cs` — пойми таймер и счёт
- [ ] Попробуй изменить баланс через `BallSettings`

## 6.2 Как добавить новый геймплей

### Пример: добавить звук при ударе о защитника

**Шаг 1.** Найди место в коде, где происходит удар

```csharp
// BallMotion.cs, метод ResolveHit
if (layer == PhysicsLayers.DefenderId)
{
    defender.HandleBallContact(this, hit);
    _bus.Publish(new BallContactEvent(BallContactKind.Defender, ...));
    return;
}
```

**Шаг 2.** Если нужно новое событие — создай его

```csharp
// Futboloid.Core/Bus/Events/DefenderHitEvent.cs
public readonly struct DefenderHitEvent
{
    public int DefenderSlotId { get; }
    public DefenderHitEvent(int slotId) => DefenderSlotId = slotId;
}
```

**Шаг 3.** Публикуй событие

```csharp
// В BallMotion
_bus.Publish(new DefenderHitEvent(defender.SlotId));
```

**Шаг 4.** Подпишись в AudioService

```csharp
// В конструкторе AudioService
_subscriptions.Add(_bus.Subscribe<DefenderHitEvent>(OnDefenderHit));

// Метод
private void OnDefenderHit(DefenderHitEvent e)
{
    _audio.Play(AudioCatalog.Ids.DefenderHit);
}
```

**Шаг 5.** Добавь звук в AudioCatalog

| Поле | Значение |
|------|----------|
| Id | `DefenderHit` |
| Channel | `GameplaySfx` |
| Clips | Перетащи файлы |
| Mixer Group | `ball` |

## 6.3 Как добавить новую UI-страницу

### Пример: добавить экран настроек

**Шаг 1.** Создай View

```csharp
// Futboloid.UI/Views/Settings/SettingsView.cs
public class SettingsView : MonoBehaviour
{
    [SerializeField] private Slider musicSlider;
    [SerializeField] private Slider sfxSlider;

    public void Show() { gameObject.SetActive(true); }
    public void Hide() { gameObject.SetActive(false); }
    public void OnMusicSliderChanged(float value) { /* ... */ }
    public void OnSfxSliderChanged(float value) { /* ... */ }
}
```

**Шаг 2.** Зарегистрируй в DI

```csharp
// В AppScopeExtensions или GameScopeExtensions
builder.RegisterComponentInScene<SettingsView>(gameScene);
```

**Шаг 3.** Подключи к навигации

```csharp
// В OverlayStateController
case NavigationState.Settings:
    settingsView.Show();
    break;
```

**Шаг 4.** Добавь состояние навигации

```csharp
// В NavigationState.cs
public enum NavigationState
{
    MainMenu,
    OnField,
    Pause,
    Tournament,
    Settings  // новое
}
```

## 6.4 Типичные ошибки и как их избегать

### Ошибка 1: Циклические зависимости

```
❌ Gameplay зависит от UI
✅ Связь только через события в Core
```

**Как избежать:** Если тебе нужно показать UI из Gameplay — не делай `using Futboloid.UI`. Вместо этого опубликуй событие.

### Ошибка 2: Забытые подписки

```
❌ Подписался на событие, но не отписался
✅ IDisposable автоматически удаляет подписчика при Dispose()
```

**Как избежать:** Всегда используй `_subscriptions.Add(_bus.Subscribe<...>(...))` — список автоматически очищается в `Dispose()`.

### Ошибка 3: new() вместо DI

```
❌ var bus = new GameEventBus();
✅ var bus = ...  // приходит из конструктора
```

**Как избежать:** Никогда не создавай сервисы через `new()`. Всё приходит из DI-контейнера.

### Ошибка 4: События как классы

```
❌ public class GoalScoredEvent { public bool IsPlayerGoal; }
✅ public readonly struct GoalScoredEvent { public bool IsPlayerGoal { get; } }
```

**Как избежать:** События — всегда `readonly struct`.

### Ошибка 5: Прямой вызов AudioSource

```
❌ GetComponent<AudioSource>().Play();
✅ _audioManager.Play("SoundId");
```

**Как избежать:** Всегда через `IAudioManager`. Никогда напрямую.
