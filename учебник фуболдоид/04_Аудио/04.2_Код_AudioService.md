## 🎯 Зачем нужен AudioService?

**AudioService** — это «переводчик» между событиями шины и звуками. Он:
1. Слушает шину событий
2. Решает, какой звук воспроизвести
3. Вызывает `IAudioManager.Play()`

## 🔍 Пошаговый разбор кода

### 1. Конструктор

```csharp
public sealed class AudioService : IDisposable
{
    private readonly IGameEventBus _bus;
    private readonly IAudioManager _audio;
    private readonly List<IDisposable> _subscriptions = new();
    private int _lastRunLevel = -1;
    private bool _onField;

    public AudioService(IGameEventBus bus, IAudioManager audio)
    {
        // Подписываемся на ВСЕ события в конструкторе
        _subscriptions.Add(_bus.Subscribe<BallContactEvent>(OnBallContact));
        _subscriptions.Add(_bus.Subscribe<GoalScoredEvent>(OnGoalScored));
        _subscriptions.Add(_bus.Subscribe<MatchStartedEvent>(OnMatchStarted));
        _subscriptions.Add(_bus.Subscribe<MatchEndedEvent>(OnMatchEnded));
        _subscriptions.Add(_bus.Subscribe<NavigationChangedEvent>(OnNavigationChanged));
        _subscriptions.Add(_bus.Subscribe<PitchPhaseChangedEvent>(OnPitchPhaseChanged));
        _subscriptions.Add(_bus.Subscribe<PerkPickedEvent>(OnPerkPicked));
        _subscriptions.Add(_bus.Subscribe<RunProgressionUpdatedEvent>(OnRunProgressionUpdated));
    }
}
```

**Зачем 8 подписок?**
- `BallContactEvent` — удары мяча
- `GoalScoredEvent` — голы
- `MatchStartedEvent` — начало матча
- `MatchEndedEvent` — конец матча
- `NavigationChangedEvent` — смена экрана
- `PitchPhaseChangedEvent` — смена фазы поля
- `PerkPickedEvent` — выбор перка
- `RunProgressionUpdatedEvent` — новый уровень

> **Правило:** Все подписки в конструкторе. НИКОГДА не в `Awake()` или `Start()`.

### 2. Метод OnBallContact() — удары мяча

```csharp
private void OnBallContact(BallContactEvent e)
{
    switch (e.Kind)
    {
        case BallContactKind.Wall:
            _audio.Play(AudioCatalog.Ids.BallHit);
            break;
        case BallContactKind.PlayerKeeper:
        case BallContactKind.Defender:
            _audio.Play(AudioCatalog.Ids.BallHitMan);
            break;
    }
}
```

**Почему `switch`?**
- Разные звуки для разных типов столкновений
- **Альтернатива:** Если-else — но switch чище

### 3. Метод OnGoalScored() — голы

```csharp
private void OnGoalScored(GoalScoredEvent e)
{
    // Игрок забил — один звук
    // Противник забил — другой звук
    _audio.Play(e.IsPlayerGoal
        ? AudioCatalog.Ids.GoalScored
        : AudioCatalog.Ids.GoalConceded);
}
```

**Почему тернарный оператор?**
- Кратко и понятно
- **Альтернатива:** if-else — но тернарный короче

### 4. Метод OnNavigationChanged() — навигация

```csharp
private void OnNavigationChanged(NavigationChangedEvent e)
{
    _onField = e.Current == NavigationState.OnField;

    // Нужно ли поставить музыку на паузу?
    if (ShouldPauseMusic(e))
    {
        _audio.PauseMusic();
    }
    // Возврат на поле?
    else if (e.Current == NavigationState.OnField)
    {
        HandleFieldMusic(e);
    }

    // Восстановить музыку при выходе из паузы
    if (e.Current != NavigationState.Pause
        && e.Previous == NavigationState.Pause && _onField)
    {
        _audio.ResumeMusic();
    }
}
```

**Разбор логики:**

| Сценарий | Действие |
|----------|----------|
| OnField → Pause | `_audio.PauseMusic()` |
| OnField → MainMenu | `_audio.PauseMusic()` |
| Pause → OnField | `_audio.ResumeMusic()` |
| MainMenu → OnField | `_audio.Play(AudioCatalog.Ids.MusicMatch)` |

### 5. Метод HandleFieldMusic() — музыка на поле

```csharp
private void HandleFieldMusic(NavigationChangedEvent e)
{
    if (!_audio.IsPlaying(AudioCatalog.Ids.MusicMatch))
        _audio.Play(AudioCatalog.Ids.MusicMatch);
}
```

**Почему `IsPlaying()`?**
- Чтобы не играть музыку дважды
- **Альтернатива:** Флаг `_musicPlaying` — но `IsPlaying()` надёжнее

---

### 6. Метод OnRunProgressionUpdated() — новый уровень

```csharp
private void OnRunProgressionUpdated(RunProgressionUpdatedEvent e)
{
    // Проверяем, вырос ли уровень
    if (_lastRunLevel >= 0 && e.RunLevel > _lastRunLevel)
        _audio.Play(AudioCatalog.Ids.LevelUp);

    _lastRunLevel = e.RunLevel;
}
```

**Почему `_lastRunLevel >= 0`?**
- Первый раз уровень = 0, не играем звук
- **Зачем?** Чтобы не играть звук при старте

---

### 7. Метод Dispose() — очистка

```csharp
public void Dispose()
{
    foreach (var subscription in _subscriptions)
        subscription.Dispose();

    _subscriptions.Clear();
}
```

**Почему `Dispose()`?**
- Удаляет все подписки
- **Зачем?** Чтобы не было утечек памяти

> **Правило:** Всегда вызывай `Dispose()` при уничтожении объекта.

## ✅ Шаблон для копирования

```csharp
public class MyAudioService : IDisposable
{
    private readonly IGameEventBus _bus;
    private readonly IMyAudioManager _audio;
    private readonly List<IDisposable> _subscriptions = new();
    
    public MyAudioService(IGameEventBus bus, IMyAudioManager audio)
    {
        _bus = bus;
        _audio = audio;
        
        // Подписываемся на события
        _subscriptions.Add(_bus.Subscribe<MyEvent>(OnMyEvent));
    }
    
    private void OnMyEvent(MyEvent e)
    {
        _audio.Play(e.SoundId);
    }
    
    public void Dispose()
    {
        foreach (var sub in _subscriptions)
            sub.Dispose();
        _subscriptions.Clear();
    }
}
```

## 🚫 Антипаттерны

### 1. Забывать Dispose()

```csharp
// ❌ ПЛОХО — утечка памяти
public class MyService
{
    public MyService(IGameEventBus bus)
    {
        bus.Subscribe<MyEvent>(OnEvent);  // Не сохраняем IDisposable!
    }
}

// ✅ ХОРОШО — сохраняем и удаляем
public class MyService : IDisposable
{
    private readonly IDisposable _subscription;
    
    public MyService(IGameEventBus bus)
    {
        _subscription = bus.Subscribe<MyEvent>(OnEvent);
    }
    
    public void Dispose()
    {
        _subscription.Dispose();
    }
}
```

### 2. Играть звук без проверки

```csharp
// ❌ ПЛОХО — играем каждый кадр
void Update()
{
    _audio.Play("Sound");  // Каждый кадр!
}

// ✅ ХОРОШО — только при событии
private void OnMyEvent(MyEvent e)
{
    _audio.Play(e.SoundId);  // Только при событии
}
```

### 3. Забывать IsPlaying()

```csharp
// ❌ ПЛОХО — играем дважды
_audio.Play("Music");
_audio.Play("Music");  // Дважды!

// ✅ ХОРОШО — проверяем
if (!_audio.IsPlaying("Music"))
    _audio.Play("Music");
```
