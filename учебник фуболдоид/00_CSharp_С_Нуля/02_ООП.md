### Шаг 16: Создание класса

```csharp
// Определение класса
class Defender
{
```

### Шаг 18: Конструктор

```csharp
class Defender
{
    private string _name;
    private int _health;
    private float _speed;
    
    // Конструктор — вызывается при создании объекта
    public Defender(string name, int health, float speed)
    {
        _name = name;
        _health = health;
        _speed = speed;
    }
}

// Использование
Defender defender = new Defender("Защитник1", 100, 5f);
```

**Почему конструктор?**
- Инициализирует объект при создании
- Не нужно писать `defender._name = "..."` после создания
- Гарантирует, что объект всегда валиден

**Задание:** Добавь конструктор в класс `Ball`.

### Шаг 19: Свойства (get/set)

```csharp
class Defender
{
    private string _name;
    private int _health;
    
    // Свойство — способ доступа к полю
    public string Name
    {
        get { return _name; }
        set { _name = value; }
    }
    
    public int Health
    {
        get { return _health; }
        private set { _health = value; } // Только для чтения извне
    }
}

// Использование
defender.Name = "Защитник1";  // Можно менять
// defender.Health = 50;      // Ошибка! Health только для чтения
```

**Почему свойства?**
- Позволяют контролировать доступ к полям
- `private set` — только класс может менять
- `get` — всегда доступен

### Шаг 24: Наследование

```csharp
// Базовый класс
class Entity
{
    public string Name { get; set; }
    public int Health { get; set; }
    
    public virtual void TakeDamage(int damage)
    {
        Health -= damage;
    }

// Наследник

    public float Speed { get; set; }
```

**Почему наследование?**
- Позволяет переиспользовать код
- Создаёт иерархию классов
- `Defender` — это `Entity`

### Шаг 25: Переопределение методов

```csharp
class Entity
{
    public string Name { get; set; }
    public int Health { get; set; }
    
    public virtual void TakeDamage(int damage)
    {
        Health -= damage;
    }
}

class Defender : Entity
{
    // Переопределение метода
    public override void TakeDamage(int damage)
    {
        // Защитник получает меньше урона
        base.TakeDamage(damage / 2);
    }
}

class Goalkeeper : Entity
{
    // Переопределение метода
    public override void TakeDamage(int damage)
    {
        // Вратарь получает ещё меньше урона
        base.TakeDamage(damage / 3);
    }
}

// Использование
Entity defender = new Defender();
defender.TakeDamage(100); // Получит 50 урона
```

**Почему `override`?**
- Позволяет изменить поведение базового класса
- `virtual` в базовом классе — разрешает переопределение
- `override` в наследнике — заменяет поведение

**Задание:** Переопредели `Move()` для разных сущностей.

**Почему интерфейсы?**

- Нельзя наследовать от нескольких классов

### Шаг 28: Полиморфизм

```csharp
class Entity
{
    public string Name { get; set; }
    public int Health { get; set; }
    
    public virtual void TakeDamage(int damage)
    {
        Health -= damage;
    }
}

class Defender : Entity
{
    public override void TakeDamage(int damage)
    {
        Health -= damage / 2;
    }
}

class Goalkeeper : Entity
{
    public override void TakeDamage(int damage)
    {
        Health -= damage / 3;
    }
}

// Полиформизм — один интерфейс, разное поведение
List<Entity> entities = new List<Entity>();
entities.Add(new Defender());
entities.Add(new Goalkeeper());

foreach (Entity entity in entities)
{
    entity.TakeDamage(100); // Каждый обработает по-своему
}
```

**Почему полиморфизм?**
- Один метод, разное поведение
- Позволяет работать с разными типами одинаково
- Делает код расширяемым

**Задание:** Создай список мячей и вызови `Move()` для каждого.

### Шаг 29: Абстрактные классы

```csharp
// Абстрактный класс — нельзя создать экземпляр
abstract class Entity
{
    public string Name { get; set; }
    public int Health { get; set; }
    
    // Абстрактный метод — обязывает наследника реализовать
    public abstract void TakeDamage(int damage);
    
    // Обычный метод — можно переопределить
    public void Heal(int amount)
    {
        Health += amount;
    }
}

class Defender : Entity
{
    public override void TakeDamage(int damage)
    {
        Health -= damage / 2;
    }
}

// Entity entity = new Entity(); // Ошибка!
Defender defender = new Defender(); // OK
```

**Почему абстрактные классы?**
- Нельзя создать экземпляр
- Обязывает наследников реализовать методы
- Позволяет добавить общую логику

**Задание:** Создай абстрактный класс `GameObject`.

### Шаг 30: Композиция vs Наследование

```csharp
// Наследование — "является"
class Defender : Entity { }  // Защитник является сущностью

// Композиция — "имеет"
class Ball
{
    private Vector2 _position;
    private float _speed;
    
    // Имеет физическое тело
    private PhysicsBody _physics;
}

class PhysicsBody
{
    public Vector2 Position { get; set; }
    public Vector2 Velocity { get; set; }
}

// Использование
Ball ball = new Ball();
ball._physics.Position = new Vector2(0, 0);
```

**Почему композиция?**
- Более гибкая, чем наследование
- Можно менять поведение во время выполнения
- Избегаем глубоких иерархий

**Задание:** Добавь `PhysicsBody` в класс `Ball`.
