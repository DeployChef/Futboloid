### Шаг 76: DI — что такое Dependency Injection

```csharp
// Без DI — создаёшь сам
public class BallMotion
{
    private IGameEventBus _bus;
    
    public BallMotion()
    {
        _bus = new GameEventBus(); // ❌ Жёсткая связь
    }
}

// С DI — получаешь извне
public class BallMotion(IGameEventBus bus)
{
    private readonly IGameEventBus _bus = bus; // ✅ Гибкая связь
}
```

**Почему DI?**
- Не создаёшь объекты через `new()`
- Получаешь зависимости из DI-контейнера
- Легко тестировать и менять

### Шаг 77: DI — регистрация в VContainer

```csharp
public class RootLifetimeScope : LifetimeScope
{
    protected override void Configure(IContainerBuilder builder)
    {
        // Singleton — один экземпляр на всё время
        builder.Register<IGameEventBus, GameEventBus>(Lifetime.Singleton);
        builder.Register<IAudioManager, AudioManager>(Lifetime.Singleton);
        
        // Transient — новый каждый раз
        builder.Register<DefenderLogic>(Lifetime.Transient);
        
        // Из сцены
        builder.RegisterComponentInScene<BallView>(gameObject.scene);
    }
}
```

**Почему разные Lifetime?**
- `Singleton` — один экземпляр на всё время
- `Transient` — новый каждый раз
- `RegisterComponentInScene` — берёт из сцены

**Задание:** Зарегистрируй `MatchFlow` как Singleton.

### Шаг 79: Event Bus — создание события

```csharp
// Событие — структура (лёгкая, в стеке)
public readonly struct BallContactEvent
{
    public BallContactKind Kind { get; }
    public Vector2 ContactPoint { get; }
    public Vector2 Normal { get; }
    public float Speed { get; }
    
    public BallContactEvent(BallContactKind kind, Vector2 point, Vector2 normal, float speed)
    {
        Kind = kind;
        ContactPoint = point;
        Normal = normal;
        Speed = speed;
    }
}
```

**Почему `readonly struct`?**
- **Производительность** — в стеке, без аллокаций
- **Неизменяемость** — нельзя изменить после создания
- **Безопасность** — никто не подменит данные

public class AudioService

    private readonly List<IDisposable> _subscriptions = new();

- Возвращает объект для удаления подписки
- Позволяет отписаться при `Dispose()`
- Предотвращает утечки памяти

### Шаг 82: Event Bus — отписка

```csharp
public class AudioService : IDisposable
{
    private readonly IGameEventBus _bus;
    private readonly List<IDisposable> _subscriptions = new();
    
    public AudioService(IGameEventBus bus)
    {
        _bus = bus;
        _subscriptions.Add(_bus.Subscribe<BallContactEvent>(OnBallContact));
    }
    
    public void Dispose()
    {
        foreach (var subscription in _subscriptions)
            subscription.Dispose();
        _subscriptions.Clear();
    }
}
```

**Почему `Dispose`?**
- Удаляет все подписки
- Предотвращает утечки памяти
- Вызывается при уничтожении объекта

**Задание:** Добавь `Dispose()` в AudioService.

### Шаг 86: FSM — PitchStateMachine

```csharp
public class PitchStateMachine
{
    private PitchPhase _currentPhase;
    private readonly IGameEventBus _bus;
    
    public PitchPhase CurrentPhase => _currentPhase;
    
    public PitchStateMachine(IGameEventBus bus)
    {
        _bus = bus;
        _currentPhase = PitchPhase.KickoffWait;
    }
    
    public void SetPhase(PitchPhase phase)
    {
        _currentPhase = phase;
        _bus.Publish(new PitchPhaseChangedEvent(phase));
    }
}
```

**Почему `Publish` при смене фазы?**
- Сообщает всем подписчикам о смене
- MatchFlow может поставить таймер на паузу
- AudioService может изменить музыку

### Шаг 90: Audio — маппинг событий

```csharp
public class AudioService : IDisposable
{
    private readonly IGameEventBus _bus;
    private readonly IAudioManager _audio;
    
    public AudioService(IGameEventBus bus, IAudioManager audio)
    {
        _bus = bus;
        _audio = audio;
        
        _bus.Subscribe<BallContactEvent>(OnBallContact);
        _bus.Subscribe<GoalScoredEvent>(OnGoalScored);
    }
    
    private void OnBallContact(BallContactEvent e)
    {
        switch (e.Kind)
        {
            case BallContactKind.Wall:
                _audio.Play(AudioCatalog.Ids.BallHit);
                break;
            case BallContactKind.Defender:
                _audio.Play(AudioCatalog.Ids.BallHitMan);
                break;
        }
    }
    
    private void OnGoalScored(GoalScoredEvent e)
    {
        _audio.Play(e.IsPlayerGoal
            ? AudioCatalog.Ids.GoalScored
            : AudioCatalog.Ids.GoalConceded);
    }
}
```

**Почему `switch`?**
- Разные звуки для разных типов столкновений
- Читаемо и понятно
- Легко добавлять новые случаи

**Задание:** Добавь звук для `PerkPickedEvent`.
