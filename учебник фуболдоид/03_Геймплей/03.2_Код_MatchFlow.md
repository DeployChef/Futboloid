## 🎯 Зачем нужен MatchFlow?

**MatchFlow** — это «секундометр» матча. Он:
1. Считает время до конца матча
2. Ведёт счёт (игрок vs противник)
3. Управляет фазами матча
4. Публикует события (таймер, гол, конец матча)

## 🔍 Пошаговый разбор кода

### 1. Конструктор

```csharp
public MatchFlow(
    IGameEventBus bus,
    GameplaySettings settings)
{
    _bus = bus;
    _settings = settings;
    _totalDurationSeconds = settings.MatchDurationSeconds;
    RemainingSeconds = _totalDurationSeconds;
    
    // Подписываемся на события
    _bus.Subscribe<GoalScoredEvent>(OnGoalScored);
    _bus.Subscribe<NavigationChangedEvent>(OnNavigationChanged);
    _bus.Subscribe<PitchPhaseChangedEvent>(OnPitchPhaseChanged);
    _bus.Subscribe<MatchTimeAdjustedEvent>(OnTimeAdjusted);
    _bus.Subscribe<BallServedEvent>(OnBallServed);
}
```

**Зачем 5 подписок?**
- `GoalScoredEvent` — обновить счёт
- `NavigationChangedEvent` — пауза при смене экрана
- `PitchPhaseChangedEvent` — пауза при выборе бонуса
- `MatchTimeAdjustedEvent` — бонус/штраф времени
- `BallServedEvent` — начать таймер

> **Правило:** Все подписки в конструкторе. НИКОГДА не в `Awake()` или `Start()`.

### 2. Метод RunTimerLoopAsync() — таймер

```csharp
private async UniTaskVoid RunTimerLoopAsync(CancellationToken ct)
{
    while (!ct.IsCancellationRequested && !_matchEnded && RemainingSeconds > 0f)
    {
        await UniTask.Yield(PlayerLoopTiming.Update, ct);

        if (!_onField || _pauseTimer)
            continue;

        RemainingSeconds = Mathf.Max(0f, RemainingSeconds - Time.deltaTime);
        PublishTimer();

        if (RemainingSeconds <= 0f)
            EndMatchByTime();
    }
}
```

**Разбор по шагам:**

| Шаг | Что делает | Зачем |
|-----|-----------|-------|
| 1 | `!ct.IsCancellationRequested` | Проверяем, не отменён ли таймер |
| 2 | `!_matchEnded` | Проверяем, не окончен ли матч |
| 3 | `RemainingSeconds > 0f` | Проверяем, не истёк ли таймер |
| 4 | `await UniTask.Yield()` | Ждём следующего кадра |
| 5 | `!_onField || _pauseTimer` | Пропускаем, если не на поле или пауза |
| 6 | `RemainingSeconds -= Time.deltaTime` | Уменьшаем таймер |
| 7 | `PublishTimer()` | Публикуем событие изменения таймера |
| 8 | `EndMatchByTime()` | Завершаем матч |

> **Важно:** `UniTask.Yield(PlayerLoopTiming.Update, ct)` — ждём следующего кадра, но учитываем паузу.

### 3. Метод OnGoalScored() — обработка гола

```csharp
private void OnGoalScored(GoalScoredEvent e)
{
    if (e.IsPlayerGoal)
        PlayerScore++;
    else
        OpponentScore++;
}
```

**Почему так просто?**
- MatchFlow только считает — не решает, кто забил
- Решение принимает `BallMotion`
- **Зачем?** Разделение ответственности

### 4. Метод OnPitchPhaseChanged() — обработка фаз

```csharp
private void OnPitchPhaseChanged(PitchPhaseChangedEvent e)
{
    // Пауза таймера при выборе бонуса
    _pauseTimer = e.Phase == PitchPhase.BonusPick;

    // Остановка матча при завершении
    if (e.Phase == PitchPhase.MatchEnded)
    {
        _matchEnded = true;
        StopTimerLoop();
    }
}
```

**Почему `_pauseTimer`?**
- При выборе бонуса таймер должен остановиться
- При возврате к игре — продолжиться
- **Альтернатива:** Останавливать и запускать таймер — но это сложнее

---

### 5. Метод EndMatch() — завершение матча

```csharp
public void EndMatch(bool won, string reason, float duration)
{
    _matchEnded = true;
    StopTimerLoop();
    
    var event = new MatchEndedEvent(PlayerScore, OpponentScore, won, reason, duration);
    _bus.Publish(event);
}
```

**Почему `StopTimerLoop()`?**
- Останавливает `CancellationTokenSource`
- `RunTimerLoopAsync()` завершается
- **Зачем?** Чтобы не было утечек памяти

## ✅ Шаблон для копирования

```csharp
public class MyTimer
{
    private float _remainingSeconds;
    private float _totalDurationSeconds;
    private CancellationTokenSource _cts;
    
    public MyTimer(float durationSeconds)
    {
        _totalDurationSeconds = durationSeconds;
        _remainingSeconds = durationSeconds;
    }
    
    public void Start()
    {
        _cts = new CancellationTokenSource();
        RunTimerLoopAsync(_cts.Token).Forget();
    }
    
    private async UniTaskVoid RunTimerLoopAsync(CancellationToken ct)
    {
        while (!ct.IsCancellationRequested && _remainingSeconds > 0f)
        {
            await UniTask.Yield(PlayerLoopTiming.Update, ct);
            _remainingSeconds = Mathf.Max(0f, _remainingSeconds - Time.deltaTime);
            
            if (_remainingSeconds <= 0f)
                EndTimer();
        }
    }
    
    public void Stop()
    {
        _cts?.Cancel();
        _cts?.Dispose();
    }
    
    private void EndTimer()
    {
        Stop();
        // Логика завершения
    }
}
```

## 🚫 Антипаттерны

### 1. Забывать отменять CancellationToken

```csharp
// ❌ ПЛОХО — утечка памяти
private void Start()
{
    var cts = new CancellationTokenSource();
    RunTimerLoopAsync(cts.Token).Forget();
}

// ✅ ХОРОШО — сохраняем и отменяем
private CancellationTokenSource _cts;

private void Start()
{
    _cts = new CancellationTokenSource();
    RunTimerLoopAsync(_cts.Token).Forget();
}

private void Stop()
{
    _cts?.Cancel();
    _cts?.Dispose();
}
```

### 2. Использовать Time.deltaTime в Update()

```csharp
// ❌ ПЛОХО — зависит от FPS
void Update()
{
    _remainingSeconds -= 1f;  // Каждый кадр!
}

// ✅ ХОРОШО — Time.deltaTime
private async UniTaskVoid RunTimerLoopAsync(CancellationToken ct)
{
    while (!ct.IsCancellationRequested)
    {
        await UniTask.Yield(PlayerLoopTiming.Update, ct);
        _remainingSeconds -= Time.deltaTime;
    }
}
```

### 3. Забывать StopTimerLoop()

```csharp
// ❌ ПЛОХО — таймер продолжается
public void EndMatch()
{
    _matchEnded = true;
    // Забыли остановить таймер!
}

// ✅ ХОРОШО — останавливаем таймер
public void EndMatch()
{
    _matchEnded = true;
    StopTimerLoop();
}
```
