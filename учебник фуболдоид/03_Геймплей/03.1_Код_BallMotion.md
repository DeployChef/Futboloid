## 🎯 Зачем нужен BallMotion?

**BallMotion** — это мозг мяча. Он:
1. Двигает мяч каждый кадр
2. Проверяет столкновения
3. Публикует события (удары, голы)
4. Управляет скоростью и направлением

### 2. Метод Tick() — основной цикл

```csharp
public void Tick(float deltaTime)
{
    if (_holdAnchor != null)
    {
        Position = _holdAnchor.WorldPosition;
        return;
    }

    if (!InPlay)
        return;

    var distance = Speed * deltaTime;
    var castDistance = distance + _settings.Skin;

    // Raycast — проверяем столкновения впереди
    var hitCount = Physics2D.CircleCast(
        Position,
        _settings.Radius,
        Direction,
        _ballContactFilter,
        _castHits,
        castDistance);

    if (hitCount > 0)
        ResolveHit(_castHits[0]);
    else
        Position += Direction * distance;

    if (TryScoreGoal())
        return;

    // Замедление
    Speed = Mathf.MoveTowards(Speed, _settings.BaseSpeed,
        _settings.Deceleration * deltaTime);
}
```

**Разбор по шагам:**

| Шаг | Что делает | Зачем |
|-----|-----------|-------|
| 1 | `_holdAnchor != null` | Если мяч привязан — не двигаем |
| 2 | `!InPlay` | Если мяч не в игре — не двигаем |
| 3 | `distance = Speed * deltaTime` | Расстояние за кадр |
| 4 | `castDistance = distance + Skin` | Skin — запас, чтобы не пропустить стены |
| 5 | `Physics2D.CircleCast()` | Проверяем столкновения впереди |
| 6 | `ResolveHit()` | Обрабатываем столкновение |
| 7 | `TryScoreGoal()` | Проверяем гол |
| 8 | `Mathf.MoveTowards()` | Замедляем до базовой скорости |

> **Важно:** `Skin` — это запас пикселей, чтобы мяч не проходил сквозь стены при высокой скорости.

### 3. Метод ResolveHit() — обработка столкновений

```csharp
private void ResolveHit(RaycastHit2D hit)
{
    var layer = hit.collider.gameObject.layer;

    // Удар о вратаря
    if (layer == PhysicsLayers.KeeperId)
    {
        ReflectFromKeeperHit(hit);
        ApplyKeeperBoost();
        _bus.Publish(new BallContactEvent(BallContactKind.PlayerKeeper, ...));
        return;
    }

    // Удар о защитника
    if (layer == PhysicsLayers.DefenderId)
    {
        defender.HandleBallContact(this, hit);
        _bus.Publish(new BallContactEvent(BallContactKind.Defender, ...));
        return;
    }

    // Удар о стену
    ReflectFromHit(hit);
    ApplyWallSpeedPenalty();
    _bus.Publish(new BallContactEvent(BallContactKind.Wall, ...));
}
```

**Почему `PhysicsLayers`?**
- `PhysicsLayers.KeeperId` — слой вратаря
- `PhysicsLayers.DefenderId` — слой защитников
- `PhysicsLayers.WallId` — слой стен
- **Зачем?** Чтобы различать типы столкновений

> **Правило:** ВСЕГДА используй `PhysicsLayers` для проверки слоёв. НИКОГДА не магию чисел.

### 4. Метод TryScoreGoal() — проверка гола

```csharp
private bool TryScoreGoal()
{
    // Гол за игрока
    if (Physics2D.OverlapCircle(Position, _settings.Radius, PhysicsLayers.GoalEnemyMask) != null)
    {
        Stop();
        _bus.Publish(new GoalScoredEvent(isPlayerGoal: true));
        return true;
    }

    // Гол за противника
    if (Physics2D.OverlapCircle(Position, _settings.Radius, PhysicsLayers.GoalPlayerMask) != null)
    {
        Stop();
        _bus.Publish(new GoalScoredEvent(isPlayerGoal: false));
        return true;
    }

    return false;
}
```

**Почему `OverlapCircle`?**
- Проверяем, перекрывает ли мяч ворота
- `GoalEnemyMask` — ворота противника
- `GoalPlayerMask` — ворота игрока

> **Важно:** После гола мяч останавливается (`Stop()`) и публикуется событие.

### 5. Метод ReflectFromHit() — отскок

```csharp
private void ReflectFromHit(RaycastHit2D hit)
{
    // Отражаем направление
    Direction = Vector2.Reflect(Direction, hit.normal);
    Direction.Normalize();
}
```

**Почему `Vector2.Reflect()`?**
- Вычисляет направление отскока
- `hit.normal` — нормаль поверхности
- **Альтернатива:** Ручное вычисление — но `Reflect()` проще и быстрее

## ✅ Шаблон для копирования

```csharp
public class MyBall : MonoBehaviour
{
    [SerializeField] private float speed = 5f;
    [SerializeField] private float radius = 0.5f;
    
    private Vector2 _direction;
    private float _speed;
    
    public void Serve(Vector2 direction)
    {
        _direction = direction.normalized;
        _speed = speed;
    }
    
    private void Update()
    {
        if (_speed > 0.01f)
        {
            var distance = _speed * Time.deltaTime;
            transform.position += (Vector3)(_direction * distance);
            _speed = Mathf.MoveTowards(_speed, 0f, deceleration * Time.deltaTime);
        }
    }
}
```

## 🚫 Антипаттерны

### 1. Двигать через Transform.position в Update()

```csharp
// ❌ ПЛОХО — не учитывает физику
void Update()
{
    transform.position += direction * speed * Time.deltaTime;
}

// ✅ ХОРОШО — через Physics2D.CircleCast
public void Tick(float deltaTime)
{
    var hit = Physics2D.CircleCast(...);
    if (hit)
        ResolveHit(hit);
    else
        Position += Direction * distance;
}
```

### 2. Забывать Normalize() направление

```csharp
// ❌ ПЛОХО — скорость меняется
Direction = Vector2.Reflect(Direction, hit.normal);

// ✅ ХОРОШО — нормализуем
Direction = Vector2.Reflect(Direction, hit.normal);
Direction.Normalize();
```

### 3. Публиковать события слишком часто

```csharp
// ❌ ПЛОХО — каждый кадр
void Update()
{
    _bus.Publish(new MyEvent());
}

// ✅ ХОРОШО — только при изменении
private int _lastValue;
void Update()
{
    if (value != _lastValue)
    {
        _bus.Publish(new MyEvent(value));
        _lastValue = value;
    }
}
```
