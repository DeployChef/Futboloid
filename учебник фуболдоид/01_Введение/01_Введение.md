## 1.1 Что такое Futboloid?

**Futboloid** — это 2D-игра с мячом, защитниками и вратарём. Игрок управляет мячом, забивает голы, выбирает перки и проходит турнир.

### Ключевые механики

- 🏀 **Физика мяча** — отскоки от стен, защитников, вратаря
- ⚽ **Голы** — забивай за команду, не пропускай
- 🛡️ **Защитники** — уничтожай их мячом
- 🎁 **Перки** — выбирай бонусы между матчами
- 📈 **Прогрессия** — качай уровень, открывай новое

## 1.2 Стек технологий

```
Unity 2D
├── VContainer        — Dependency Injection
├── UniTask           — Async/Await (не блокирует поток)
├── ScriptableObject  — Данные и настройки
├── AudioMixer        — Управление звуком
└── 2D Physics        — Физика мяча
```

### Почему именно эти технологии?

| Технология | Зачем нужна | Альтернатива | Почему выбрали |
|------------|-------------|--------------|----------------|
| **VContainer** | DI-контейнер | Addressables, Zenject | Быстрее, легче, проще |
| **UniTask** | Асинхронность | Coroutine, Task | Нет аллокаций, быстрее |
| **ScriptableObject** | Данные | JSON, CSV | Встроен в Unity, горячая перезагрузка |
| **AudioMixer** | Звук | Громкость на AudioSource | Группировка, экспозируемые параметры |

## 1.4 Архитектурные паттерны

### 1. Dependency Injection (DI)

**Зачем:** Не создавай объекты через `new()`. Получай их из контейнера.

```csharp
// ❌ ПЛОХО — создаёшь сам
var bus = new GameEventBus();

// ✅ ХОРОШО — получаешь из DI
public class MyClass(IGameEventBus bus) { }
```

### 2. Event Bus (Шина событий)

**Зачем:** Компоненты общаются без прямых ссылок.

```csharp
// BallMotion публикует
_bus.Publish(new GoalScoredEvent(isPlayer: true));

// AudioService слушает
_bus.Subscribe<GoalScoredEvent>(OnGoalScored);
```

### 3. Finite State Machine (FSM)

**Зачем:** Управление фазами поля (подача, игра, пауза).

```csharp
public enum PitchPhase { KickoffWait, Simulating, BonusPick, MatchEnded }
```
