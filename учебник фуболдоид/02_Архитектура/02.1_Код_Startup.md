## 🎯 Зачем нужен Startup?

**Startup** — это «дверь» в игру. Он:
1. Запускает DI-контейнер
2. Создаёт GameDirector
3. Инициализирует игру
4. Скрывает экран загрузки

> **Правило:** Startup — ЕДИНСТВЕННОЕ место, где вызывается `new` для GameDirector. Всё остальное — через DI.

---

## 🔍 Пошаговый разбор кода

### 1. Приватные поля

```csharp
private static bool _started;  // Защита от дубля
private RootLifetimeScope _rootScope;
private CanvasGroup _loadingScreen;
```

**Зачем `static _started`?**
- Гарантирует, что `Awake()` вызовется только один раз
- Если Unity вызовет `Awake()` дважды (редко, но бывает), второй раз пропустит
- **Альтернатива:** `Application.isPlaying` — но `static bool` надёжнее

**Почему не `[SerializeField]` для `_rootScope`?**
- Потому что `RootLifetimeScope` — это компонент сцены
- Его можно перетащить в Inspector (если бы был `[SerializeField]`)

---

### 2. Awake() — начало

```csharp
private void Awake()
{
    if (_started) return;  // Защита от дубля
    _started = true;
    BootAsync().Forget();  // Запуск асинхронной инициализации
}
```

**Зачем `.Forget()`?**
- `BootAsync()` возвращает `UniTaskVoid`
- `.Forget()` говорит UniTask: «не жди завершения, не логи ошибки»
- **Альтернатива:** `await` — но тогда `Awake()` не может быть `async`
- **Почему не `Task.Run()`?** — UniTask оптимизирован для Unity

> **Антипаттерн:**
> ```csharp
> // ❌ ПЛОХО — блокирует поток
> BootAsync().Wait();
> 
> // ❌ ПЛОХО — создаёт Task (аллокации)
> Task.Run(() => BootAsync());
> ```

---

### 3. BootAsync() — основная логика

```csharp
private async UniTaskVoid BootAsync()
{
    // 1. Показываем loading screen
    _loadingScreen.alpha = 1f;
    
    // 2. Создаём DI-контейнер
    _rootScope.Build();
    
    // 3. Создаём Director и инициализируем игру
    var director = _rootScope.Container.Resolve<IGameDirector>();
    await director.InitializeGameAsync();
    
    // 4. Скрываем loading screen
    await FadeOutLoadingAsync();
}
```

**Почему `await` здесь?**
- `InitializeGameAsync()` — асинхронная инициализация
- `FadeOutLoadingAsync()` — плавное исчезновение
- `UniTaskVoid` позволяет использовать `await` без возврата значения

**Что делает `_rootScope.Build()`?**
1. Создаёт DI-контейнер VContainer
2. Регистрирует все сервисы (Root, App, Game Scope)
3. Возвращает готовый контейнер

> **Важно:** После `Build()` контейнер НЕЛЬЗЯ менять. Регистрация только до `Build()`.

---

### 4. FadeOutLoadingAsync() — плавное исчезновение

```csharp
private async UniTask FadeOutLoadingAsync()
{
    var t = 0f;
    while (t < 1f)
    {
        await UniTask.Yield();
        t += Time.unscaledDeltaTime;
        _loadingScreen.alpha = Mathf.Lerp(1f, 0f, t);
    }
    _loadingScreen.alpha = 0f;
}
```

**Зачем `Time.unscaledDeltaTime`?**
- `Time.deltaTime` — учитывает паузу (pauseScale)
- `Time.unscaledDeltaTime` — НЕ учитывает паузу
- Нам нужно, чтобы fade работал даже при паузе

**Почему `Mathf.Lerp(1f, 0f, t)`?**
- `t = 0` → `alpha = 1` (полностью видимый)
- `t = 1` → `alpha = 0` (полностью прозрачный)
- **Альтернатива:** `Mathf.MoveTowards` — но Lerp проще для понимания

## ✅ Шаблон для копирования

```csharp
public class MyStartup : MonoBehaviour
{
    [SerializeField] private LifetimeScope scope;
    [SerializeField] private CanvasGroup loadingScreen;
    
    private async UniTaskVoid BootAsync()
    {
        // 1. Показать loading screen
        loadingScreen.alpha = 1f;
        
        // 2. Создать DI-контейнер
        scope.Build();
        
        // 3. Resolve главного контроллера
        var controller = scope.Container.Resolve<IMyController>();
        await controller.InitializeAsync();
        
        // 4. Скрыть loading screen
        await FadeOutAsync();
    }
    
    private async UniTask FadeOutAsync()
    {
        var t = 0f;
        while (t < 1f)
        {
            await UniTask.Yield();
            t += Time.unscaledDeltaTime;
            loadingScreen.alpha = Mathf.Lerp(1f, 0f, t);
        }
    }
}
```

## 🚫 Антипаттерны

### 1. Создавать объекты через `new()`

```csharp
// ❌ ПЛОХО
var bus = new GameEventBus();

// ✅ ХОРОШО
var bus = _rootScope.Container.Resolve<IGameEventBus>();
```

### 2. Блокировать поток

```csharp
// ❌ ПЛОХО
BootAsync().Wait();

// ✅ ХОРОШО
await BootAsync();
```

### 3. Забывать `.Forget()`

```csharp
// ❌ ПЛОХО — компилятор ругается
BootAsync();

// ✅ ХОРОШО
BootAsync().Forget();
```
