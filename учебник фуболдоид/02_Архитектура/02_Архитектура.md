## 2.1 Структура папок

### Главное правило

**Всё своё** — в `Assets/_Projects/`. Префикс `_` поднимает папку вверх в Project window.

```
Assets/
├── _Projects/           ← ВСЁ своё
│   ├── Code/            ← C# + asmdef (6 сборок)
│   ├── Scenes/          ← Unity-сцены (.unity)
│   ├── Resources/       ← Грузим через Resources.Load
│   │   ├── Prefabs/     ← prefab'ы
│   │   └── Data/        ← ScriptableObject
│   ├── Art/             ← Sprites, animations
│   ├── Audio/           ← Music, SFX
│   └── Prefabs/         ← prefab'ы для сцен
├── Plugins/             ← Сторонние пакеты
└── Settings/            ← URP, Input
```

> **НИЧЕГО** не клади в корень `Assets/` — никаких разрозненных `Scripts/`, `Sprites/`.

## 2.2 Сборки (asmdef) и зависимости

Проект разделён на **6 сборок** (asmdef). Каждая — отдельный модуль.

```
Futboloid.Editor  ----->  Futboloid.Main  ----->  Futboloid.Gameplay
                                                              |
Futboloid.UI  ------------------------------------------------+
                                                              |
Futboloid.Core  ----------------------------------------------+
                                                              |
Futboloid.Utilities  (ни от чего не зависит)
```

### Описание сборок

| Сборка | Содержимое | Зависит от |
|--------|-----------|------------|
| `Utilities` | Extensions, хелперы | — |
| `Core` | Интерфейсы, шина, DI, навигация | Utilities |
| `UI` | Виджеты, HUD, меню | Core |
| `Gameplay` | Мяч, вратарь, защитники | Core |
| `Main` | Startup, DI, GameDirector | Core, UI, Gameplay |
| `Editor` | Custom inspectors | Main (Editor only) |

> **ВАЖНО:** НЕТ циклических зависимостей! Gameplay НЕ должен ссылаться на UI.

## 2.4 DI-контейнер VContainer

### Что такое DI?

**Dependency Injection** — объект получает зависимости «извне», а не создаёт их сам.

**Зачем:**
- Не нужно `new()` — контейнер делает это за нас
- Легко заменять реализации
- Контролируем время жизни (Singleton, Transient)

### VContainer — DI-контейнер для Unity

```
RootLifetimeScope (Root.unity)
├── Регистрирует: GameDirector, AudioManager, EventBus
└── При переходе на Game: создает GameScope
    ├── Регистрирует: BallMotion, MatchFlow
    └── Регистрирует: BallView (из сцены)
```

### Как регистрируются сервисы

```csharp
// Singleton — один экземпляр на всё время
builder.Register<MatchFlow>(Lifetime.Singleton);

// Transient — новый каждый раз при запросе
builder.Register<DefenderLogic>(Lifetime.Transient);

// Компонент из сцены
builder.RegisterComponentInScene<BallView>(gameScene);
```

## 2.5 Шина событий (Event Bus)

### Что такое Event Bus?

**Шина событий** — «почтовый ящик» между объектами. Кто-то **публикует**,
кто-то **подписывается** — и они общаются, не зная друг о друге.

```
BallMotion  -- BallContactEvent -->  AudioService
   |
   |  GoalScoredEvent
   v
MatchFlow
```

### Почему события — это структуры (struct)?

```csharp
public readonly struct GoalScoredEvent
{
    public bool IsPlayerGoal { get; }
}
```

**Причины:**
1. **Производительность** — структуры в стеке, без аллокаций
2. **Неизменяемость** — `readonly struct` нельзя изменить
3. **Безопасность** — никто не подменит данные

## 2.6 Оверлеи и навигация

### OverlayStateController

Управляет видимостью экранов (меню, пауза, поле).

```
OverlayStateController
├── MainMenu (Canvas)
├── Pause (Canvas)
├── Tournament (Canvas)
└── OnField (Canvas — игровой экран)
```

### Состояния навигации

```csharp
public enum NavigationState
{
    MainMenu,      // Главное меню
    OnField,       // Игровое поле
    Pause,         // Пауза
    Tournament     // Турнирная таблица
}
```

## 2.7 FSM (Машина состояний поля)

### PitchPhase — фазы игрового поля

```csharp
public enum PitchPhase
{
    KickoffWait,    // Ожидание подачи мяча
    Simulating,     // Мяч в игре
    Reshuffle,      // Защитники перегруппированы
    BonusPick,      // Игрок выбирает перк
    MatchEnded      // Матч окончен
}
```

### Жизненный цикл фазы

```
KickoffWait → Simulating → BonusPick → Simulating → ... → MatchEnded
               ↑                              ↓
          DefenderDestroyed            PerkPicked
```
