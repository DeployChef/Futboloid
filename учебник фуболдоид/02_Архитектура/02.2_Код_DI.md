## 🎯 Зачем нужен DI?

**Dependency Injection** — объект получает зависимости «извне», а не создаёт их сам.

### Проблема без DI

```csharp
public class BallMotion
{
    private IGameEventBus _bus;
    private IAudioManager _audio;
    
    public BallMotion()
    {
        // ❌ ЖЁСТКАЯ СВЯЗЬ — трудно тестировать, трудно менять
        _bus = new GameEventBus();
        _audio = new AudioManager();
    }
}
```

### Решение с DI

```csharp
public class BallMotion(IGameEventBus bus, IAudioManager audio)
{
    // ✅ ГИБКАЯ СВЯЗЬ — легко тестировать, легко менять
    private readonly IGameEventBus _bus = bus;
    private readonly IAudioManager _audio = audio;
}
```

## 🔍 Что такое LifetimeScope?

**LifetimeScope** — Unity-объект, который содержит регистрацию сервисов.

### Иерархия LifetimeScope

```
RootLifetimeScope (Root.unity)
├── Регистрирует: GameDirector, AudioManager, EventBus
└── При переходе на Game: создает GameScope
    ├── Регистрирует: BallMotion, MatchFlow
    └── Регистрирует: BallView (из сцены)
```

### Зачем иерархия?

1. **RootLifetimeScope** — сервисы на всё время игры
2. **GameScope** — сервисы только на время матча
3. При уходе с матча — GameScope уничтожается, освобождая память

## 🔍 Пошаговый разбор RootLifetimeScope

### 1. Структура класса

```csharp
public class RootLifetimeScope : LifetimeScope
{
    [SerializeField] private CanvasGroup _loadingScreen;
    
    protected override void Configure(IContainerBuilder builder)
    {
        // Регистрация сервисов
    }
}
```

**Почему `LifetimeScope`?**
- Наследуется от `MonoBehaviour` — можно добавить на GameObject
- Переопределяет `Configure()` — место для регистрации
- Автоматически управляет жизненным циклом

---

### 2. Метод Configure()

```csharp
protected override void Configure(IContainerBuilder builder)
{
    // 1. Шина событий
    builder.Register<IGameEventBus, GameEventBus>(Lifetime.Singleton);
    
    // 2. Аудио
    builder.RegisterComponentInScene<AudioManager>();
    
    // 3. GameDirector
    builder.Register<IGameDirector, GameDirector>(Lifetime.Singleton);
    
    // 4. AppRootState
    builder.Register<AppRootState>(Lifetime.Singleton);
}
```

**Разбор параметров:**

| Параметр | Значение | Когда использовать |
|----------|----------|-------------------|
| `Lifetime.Singleton` | Один экземпляр на всё время | EventBus, AudioManager |
| `Lifetime.Transient` | Новый каждый раз | DefenderLogic, Temporary objects |
| `RegisterComponentInScene` | Берёт из сцены | BallView, GoalkeeperView |

> **Правило:** Если сервис нужен везде — `Singleton`. Если только в матче — `GameScope.Singleton`.

---

### 3. Пример регистрации с зависимостями

```csharp
builder.Register<MatchFlow>(Lifetime.Singleton)
    .WithParameter(typeof(IGameEventBus), Resolve<IGameEventBus>());
```

**Зачем `WithParameter`?**
- Если конструктор принимает нестандартные параметры
- **Альтернатива:** Внедрить через конструктор (предпочтительно)

## 🔍 Как использовать зарегистрированные сервисы

### 1. Constructor Injection (рекомендуется)

```csharp
public class BallMotion(
    IGameEventBus bus,
    IAudioManager audio)
{
    private readonly IGameEventBus _bus = bus;
    private readonly IAudioManager _audio = audio;
}
```

**Почему конструктор?**
- Зависимости видны сразу
- Легко тестировать
- DI-контейнер сам подставит значения

---

### 2. Method Injection (редко)

```csharp
public class BallMotion
{
    [Inject]
    public void Inject(IGameEventBus bus)
    {
        _bus = bus;
    }
}
```

**Когда использовать?**
- Если зависимость необязательная
- Если конструктор слишком длинный

> **Антипаттерн:** Никогда не используй `GetComponent<T>()` для получения сервисов из DI.

---

## 📊 Метрики DI

| Метрика | Значение | Описание |
|---------|----------|----------|
| Singleton-сервисов | 10+ | EventBus, AudioManager, и др. |
| Transient-сервисов | 5+ | DefenderLogic, Temporary objects |
| Компонентов из сцены | 8+ | BallView, GoalkeeperView, и др. |
| LifetimeScope | 2 | Root, Game |

## ✅ Шаблон для копирования

```csharp
public class MyLifetimeScope : LifetimeScope
{
    protected override void Configure(IContainerBuilder builder)
    {
        // Singleton — на всё время
        builder.Register<IMyService, MyService>(Lifetime.Singleton);
        
        // Transient — новый каждый раз
        builder.Register<ITempService, TempService>(Lifetime.Transient);
        
        // Из сцены
        builder.RegisterComponentInScene<MyComponent>(gameObject.scene);
    }
}
```

## 🚫 Антипаттерны

### 1. Создавать объекты через `new()`

```csharp
// ❌ ПЛОХО
var service = new MyService();

// ✅ ХОРОШО
var service = container.Resolve<IMyService>();
```

### 2. Забывать регистрировать

```csharp
// ❌ ПЛОХО — ошибка при Resolve
// builder.Register<IMyService, MyService>(Lifetime.Singleton);

// ✅ ХОРОШО
builder.Register<IMyService, MyService>(Lifetime.Singleton);
```

### 3. Циклические зависимости

```csharp
// ❌ ПЛОХО — A зависит от B, B зависит от A
public class A(IB b) { }
public class B(IA a) { }

// ✅ ХОРОШО — через Event Bus
public class A(IGameEventBus bus) { }
public class B(IGameEventBus bus) { }
```
