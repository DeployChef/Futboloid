## 🎯 Зачем нужен Event Bus?

**Event Bus** — «почтовый ящик» между объектами. Кто-то **публикует**,
кто-то **подписывается** — и они общаются, не зная друг о друге.

### Проблема без Event Bus

```csharp
public class BallMotion
{
    private IAudioManager _audio;
    
    public BallMotion(IAudioManager audio)
    {
        _audio = audio;
    }
    
    public void OnBallHit()
    {
        _audio.Play("BallHit");  // ❌ Жёсткая связь
    }
}
```

### Решение с Event Bus

```csharp
public class BallMotion
{
    private IGameEventBus _bus;
    
    public BallMotion(IGameEventBus bus)
    {
        _bus = bus;
    }
    
    public void OnBallHit()
    {
        _bus.Publish(new BallContactEvent(...));  // ✅ Гибкая связь
    }
}

public class AudioService
{
    public AudioService(IGameEventBus bus, IAudioManager audio)
    {
        _bus.Subscribe<BallContactEvent>(OnBallContact);
    }
    
    private void OnBallContact(BallContactEvent e)
    {
        _audio.Play("BallHit");
    }
}
```

## 🔍 Пошаговый разбор IGameEventBus

### 1. Интерфейс

```csharp
public interface IGameEventBus
{
    void Publish<T>(T message) where T : struct;
    IDisposable Subscribe<T>(Action<T> handler) where T : struct;
}
```

**Почему `where T : struct`?**
- Гарантирует, что событие — структура (лёгкая, в стеке)
- Нельзя передать класс (аллокации в куче)
- **Альтернатива:** `where T : new()` — но struct надёжнее

**Почему `IDisposable Subscribe()`?**
- Возвращает объект, который удалит подписчика при `Dispose()`
- **Зачем это нужно?** Чтобы не было утечек памяти

---

### 2. Структура события

```csharp
public readonly struct BallContactEvent
{
    public BallContactKind Kind { get; }
    public Vector2 ContactPoint { get; }
    public Vector2 Normal { get; }
    public float Speed { get; }
    
    public BallContactEvent(BallContactKind kind, Vector2 point, Vector2 normal, float speed)
    {
        Kind = kind;
        ContactPoint = point;
        Normal = normal;
        Speed = speed;
    }
}
```

**Почему `readonly struct`?**
- **Производительность** — в стеке, без аллокаций
- **Неизменяемость** — нельзя изменить после создания
- **Безопасность** — никто не подменит данные

> **Антипаттерн:**
> ```csharp
> // ❌ ПЛОХО — class (аллокации в куче)
> public class BallContactEvent { public BallContactKind Kind; }
> 
> // ✅ ХОРОШО — readonly struct
> public readonly struct BallContactEvent { public BallContactKind Kind { get; } }
> ```

## 🔍 Пошаговый разбор GameEventBus

### 1. Структура класса

```csharp
public class GameEventBus : IGameEventBus
{
    private readonly Dictionary<Type, List<Delegate>> _handlers = new();
}
```

**Почему `Dictionary<Type, List<Delegate>>`?**
- `Type` — тип события (например, `typeof(BallContactEvent)`)
- `List<Delegate>` — список обработчиков для этого типа
- **Альтернатива:** `Dictionary<Type, List<Action<T>>>` — но обобщённый вариант сложнее

---

### 2. Метод Publish()

```csharp
public void Publish<T>(T message) where T : struct
{
    var type = typeof(T);
    if (!_handlers.TryGetValue(type, out var list)) return;
    
    // Делаем копию — чтобы изменения в handler не сломали итерацию
    var snapshot = list.ToArray();
    foreach (var handler in snapshot)
        ((Action<T>)handler)(message);
}
```

**Почему `snapshot = list.ToArray()`?**
- Если обработчик отпишется во время итерации — список изменится
- `ToArray()` создаёт копию, итерация не сломается
- **Альтернатива:** `foreach` с `ToList()` — но `ToArray()` быстрее

**Почему `((Action<T>)handler)(message)`?**
- `handler` — это `Delegate`, нужно привести к `Action<T>`
- **Альтернатива:** `handler.DynamicInvoke(message)` — но медленнее

---

### 3. Метод Subscribe()

```csharp
public IDisposable Subscribe<T>(Action<T> handler) where T : struct
{
    var type = typeof(T);
    if (!_handlers.TryGetValue(type, out var list))
    {
        list = new List<Delegate>();
        _handlers[type] = list;
    }
    
    list.Add(handler);
    
    return new Subscription(() => list.Remove(handler));
}
```

**Почему возвращаем `IDisposable`?**
- Позволяет отписаться при `Dispose()`
- **Зачем это нужно?** Чтобы не было утечек памяти

---

### 4. Класс Subscription

```csharp
private class Subscription : IDisposable
{
    private readonly Action _unsubscribe;
    
    public Subscription(Action unsubscribe)
    {
        _unsubscribe = unsubscribe;
    }
    
    public void Dispose()
    {
        _unsubscribe();
    }
}
```

**Почему отдельный класс?**
- Инкапсулирует логику отписки
- **Альтернатива:** `Delegate` — но `IDisposable` понятнее

## ✅ Шаблон для копирования

### 1. Создание события

```csharp
public readonly struct MyNewEvent
{
    public string Message { get; }
    public int Value { get; }
    
    public MyNewEvent(string message, int value)
    {
        Message = message;
        Value = value;
    }
}
```

### 2. Подписка

```csharp
public class MyService
{
    private readonly IGameEventBus _bus;
    private readonly IDisposable _subscription;
    
    public MyService(IGameEventBus bus)
    {
        _bus = bus;
        _subscription = _bus.Subscribe<MyNewEvent>(OnMyEvent);
    }
    
    private void OnMyEvent(MyNewEvent e)
    {
        Debug.Log(e.Message);
    }
    
    public void Dispose()
    {
        _subscription.Dispose();
    }
}
```

### 3. Публикация

```csharp
public class MyPublisher
{
    private readonly IGameEventBus _bus;
    
    public MyPublisher(IGameEventBus bus)
    {
        _bus = bus;
    }
    
    public void DoSomething()
    {
        _bus.Publish(new MyNewEvent("Hello", 42));
    }
}
```

## 🚫 Антипаттерны

### 1. События как классы

```csharp
// ❌ ПЛОХО — class (аллокации)
public class MyEvent { public string Message; }

// ✅ ХОРОШО — readonly struct
public readonly struct MyEvent { public string Message { get; } }
```

### 2. Забывать отписаться

```csharp
// ❌ ПЛОХО — утечка памяти
_bus.Subscribe<MyEvent>(OnEvent);

// ✅ ХОРОШО — сохраняем IDisposable
var sub = _bus.Subscribe<MyEvent>(OnEvent);
// ... позже
sub.Dispose();
```

### 3. Публиковать события в Update()

```csharp
// ❌ ПЛОХО — слишком часто
void Update()
{
    _bus.Publish(new MyEvent());  // Каждый кадр!
}

// ✅ ХОРОШО — только при изменении
private int _lastValue;
void Update()
{
    if (_value != _lastValue)
    {
        _bus.Publish(new MyEvent(_value));
        _lastValue = _value;
    }
}
```
